BELOW ARE A FEW HELPFUL RECURRING PATTERNS I’VE ENCOUNTERED WHILE SOLVING MONOTONIC TYPE PROBLEMS

-> While solving the problem we encounter a scenario where we have a subarray and a subproblem requires us to consistently perform an 
   operation on the least or highest value in the subarray, usually multiple times to maintain an order 
 ex:- SubarrayWithElementsGreaterThanVaryingThreshold.py

-> Most evident clue, we need to maintain an order of highest or lowest values in order 
 ex:- MaximuminASlidingWindow.js; we need to find maximum in a window but finding max is a O(n) operation, 
      maintaining monotonic order of decreasing elements makes it a O(1) operation

-> A 132 pattern sub problem where difference in values (heights) are the main focus of the problem and we need to add/remove elements
   based on this relationship

1) -> Adding or removing element from the stack/queue also gives us information which need to be used to solve few problems
 ex:- LargestRectangleinHistogram.py, we need to find the max rectangle area, now consider the scenario
  4,6,7,5; 4,6,7 forms a block; while adding increasing elements we can calculate max area; ex 4,5 -> max(5,max_area) 
  now, when we encounter a value less than the top of the stack value we need to remove it 
  while removing elements the range of index from current value and the removed element value helps us in finding the max area
  |current value index - popped values index|, essentially moving backwards

  -> Similarly, while adding elements to the que, few problems require us to store some form of information
 ex:- ApplyOperationstoMaximizeScore.py; In this - 
  A subproblem requires us to find the range for each element where it has the highest prime factor property within the subarray
  Range requries us to find the start and end value, the start value is determined while adding the element and end while popping it

2) -> If we encounter a situation where we have a monotonic decreasing stack and the new higher value requires to remove the first element
  but we need to consider all future higher values as well to

TLDR :-
1) 132 type problem, highest or lowest value needed -> this part doesn't easily jump out; would have to think deep for a bit
2) Be mindful while adding and removing elements
3) While removing element, try expanding from back, helps in formulating the solution
4) Decreasing values but future higher values also needed? Try iterating backwards and storing decreasing values
   ex:- 6,4,2,7,8,9 -> for 6 we need -> 7,8,9 but only get 7 if decreasing value left to right;
